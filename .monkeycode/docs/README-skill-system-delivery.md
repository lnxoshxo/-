## 5. 一周上手路径（可直接执行）

### Day 1：定目标
1. 用 `board-brief` 明确本年度 3-5 个核心管理目标。
2. 在 `okr-kpi-map` 里确定总经理级 O/KR 与季度 KPI。

### Day 2：定职责
1. 基于 `raci-sop` 固化 GM/DIR/MGR/SPC 的 A 角色。
2. 发布《授权矩阵》并同步升级触发条件。

### Day 3：定流程
1. 选取 3 条最高频流程先行（推荐：立项上线、故障应急、预算复盘）。
2. 按 `raci-sop` 明确时限和固定输出。

### Day 4：定模板
1. 从模板库启用 6 个必用模板：立项、验收、上线预案、故障复盘、预算、ROI。
2. 统一命名与归档目录，确保可审计。

### Day 5：定节奏
1. 启动周检/月检/季检机制。
2. 指定会议主持、参会角色、固定产出。

### Day 6：试运行
1. 挑选一个在建项目跑全链路。
2. 挑选一次历史故障做桌面演练。

### Day 7：复盘与修订
1. 输出首周复盘报告。
2. 修订不合理 KPI、流程步骤和模板字段。

## 6. 最小落地标准（30 天）
1. 100% 关键项目纳入统一里程碑与周报机制。
2. 100% 重大事件执行统一应急指挥和复盘模板。
3. 100% 预算申请可追溯到 Skill 与项目。
4. 100% 关键流程具备 RACI 与审计留痕。

## 7. 常见失败点与规避

| 失败点 | 表现 | 规避动作 |
|---|---|---|
| 只发文不执行 | 文档齐全但无人使用 | 把模板与绩效绑定 |
| 角色责任重叠 | 出问题后互相推诿 | 每个 Skill 只保留一个 A |
| 数据口径不一致 | 汇报反复解释数字 | 先固化指标字典再出报表 |
| 复盘无闭环 | 问题重复发生 | 复盘项必须有责任人和截止日 |

## 8. 快速启动命令（管理动作）
1. 发布《技能体系执行通知》。
2. 发布《RACI 与授权矩阵》。
3. 启动《周检/月检/季检例会机制》。
4. 启用《六大核心模板》。
5. 启用《季度 KPI 评分卡》。
