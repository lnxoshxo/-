## 3. 核心能力增量
1. 从“职责描述”升级为“可执行 Skill 单元”，每个 Skill 自带输入、流程、输出、KPI、风险、授权。
2. 从“单点管理”升级为“端到端闭环”，覆盖战略、项目、运维、安全、数据、预算、协同、应急。
3. 从“经验驱动”升级为“模板驱动 + 指标驱动 + 审计驱动”。
4. 从“难考核”升级为“OKR/KPI 可映射、可量化、可归因”。

## 4. 使用顺序（建议）
1. 管理层先读：`board-brief`、`notice`。
2. 负责人执行：`main`、`raci-sop`、`okr-kpi-map`。
3. 执行层落地：`execution-checklist`、`templates`、`quarterly-report-template`。
4. PMO 跟踪：`90-day-rollout-plan`、`README-delivery`。

## 5. 版本验收标准
1. 关键项目纳管率 >= 95%。
2. 核心模板使用率 >= 95%。
3. 季度 KPI 报送完整率 = 100%。
4. 重大事件复盘及时率 = 100%。
5. 预算申请 Skill 追溯率 = 100%。

## 6. 已知约束
1. 本版未包含具体系统工具配置（如流程引擎、看板工具、BI 平台实现细节）。
2. 各区域业态差异需在 `4.03.02` 和 `1.02.03` 下进行本地化参数扩展。
3. 个别 KPI 阈值需结合历史基线在 Q1 试运行后二次校准。

## 7. 下个版本计划（v1.1.0 建议）
1. 增加“区域公司差异化参数包”。
2. 增加“项目公司现场运维快速手册”。
3. 增加“自动化报表字段字典与接口映射”。
4. 输出“年度审计证据索引目录”。

## 8. 发布与生效建议
1. 由信息化部总经理签发执行通知，明确生效日期。
2. 7 天内完成总部与区域双层宣贯。
3. 30 天内完成首轮试运行与复盘修订。
4. 90 天内完成首轮季度通报与绩效联动。
