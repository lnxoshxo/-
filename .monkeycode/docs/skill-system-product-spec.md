# 物业集团信息化 Skill 系统化建设规格书（可研发版）

## 1. 建设目标
- 把 108 个 Skill 从“文档能力”升级为“系统能力”。
- 实现 Skill 的在线编排、流程驱动、权限控制、证据留痕、KPI 自动汇总。
- 支撑总部、区域、项目三级组织协同运行。

## 2. 系统定位
- 系统名称：`IGIT SkillOS`（Information Governance & IT Skill Operating System）。
- 系统角色：信息化管理中台，不替代业务系统，负责治理与流程编排。
- 核心职责：
  1. 管 Skill 定义（标准化）
  2. 管 Skill 执行（流程化）
  3. 管 Skill 结果（指标化）
  4. 管 Skill 证据（审计化）

## 3. 产品架构

### 3.1 功能域
1. Skill 字典中心
2. 流程编排中心
3. 任务与工单中心
4. 交付物与模板中心
5. KPI 与驾驶舱中心
6. 权限与组织中心
7. 风险与应急中心
8. 预算与供应商中心
9. 审计与归档中心

### 3.2 技术分层
1. 表现层：Web 管理端（总部/区域/项目多视图）
2. 业务层：Skill 引擎 + 流程引擎 + 规则引擎
3. 数据层：事务库 + 分析库 + 文档对象存储
4. 集成层：统一 API 网关 + 事件总线 + SSO

## 4. 核心对象模型

### 4.1 主对象
1. `SkillDefinition`
  - 字段：skill_code、name、domain、module、goal、input_schema、output_schema、kpi_schema、risk_schema、raci
2. `SkillInstance`
  - 字段：instance_id、skill_code、trigger_source、org_level、status、owner、start_at、due_at
3. `FlowDefinition`
  - 字段：flow_id、name、nodes、transitions、sla_policy、escalation_policy
4. `Task`
  - 字段：task_id、instance_id、node_id、assignee、priority、status、deadline
5. `Deliverable`
  - 字段：deliverable_id、instance_id、template_id、file_uri、version、approved_by
6. `KPIRecord`
  - 字段：kpi_id、skill_code、period、target_value、actual_value、source
7. `RiskIssue`
  - 字段：risk_id、instance_id、severity、impact、mitigation、owner、closure_at

### 4.2 关系
- 一个 `SkillDefinition` 对应多个 `SkillInstance`。
- 一个 `SkillInstance` 绑定一个 `FlowDefinition`。
- 一个 `SkillInstance` 产生多个 `Task`、`Deliverable`、`KPIRecord`、`RiskIssue`。

## 5. 功能规格（按模块）

### 5.1 Skill 字典中心
目标：把 108 个 Skill 结构化入库。
功能点：
1. Skill 新增/编辑/版本管理。
2. 字段校验（输入、流程、输出、KPI、风险必填）。
3. Skill 依赖关系配置（上下游 Skill）。
4. Skill 适用范围配置（业态、组织级别）。

验收标准：
- 108 个 Skill 100% 入库。
- 每个 Skill 至少绑定 1 个模板、1 个流程、1 组 KPI。

### 5.2 流程编排中心
目标：让 Skill 可自动流转。
功能点：
1. BPMN 可视化编排。
2. 节点类型：审批、执行、会签、自动任务、网关。
3. SLA 规则：超时提醒、升级、驳回。
4. 红线触发：P1 故障/超预算/高危漏洞自动升级 GM。

验收标准：
- 三条主链路可跑通：平台上线链、故障应急链、预算编制链。

### 5.3 任务与工单中心
目标：让执行动作有单可跟。
功能点：
1. 任务自动生成与分派。
2. 看板视图（总部/区域/项目）。
3. 状态流转（待办、进行中、阻塞、已完成）。
4. 协同与催办机制。

验收标准：
- 周检/月检/季检任务自动派发成功率 >= 99%。

### 5.4 交付物与模板中心
目标：模板驱动，交付可审计。
功能点：
1. 模板库管理（12 个模板起步）。
2. 一键生成文档实例。
3. 电子签核与版本留痕。
4. 自动归档到对象存储。

验收标准：
- 核心流程模板使用率 >= 95%。

### 5.5 KPI 与驾驶舱中心
目标：自动度量 Skill 执行结果。
功能点：
1. KPI 指标字典管理。
2. 自动采集（工单系统、监控系统、财务系统）。
3. 经营驾驶舱（战略/交付/稳定/安全/数据/成本六视角）。
4. 季度通报自动生成。

验收标准：
- KPI 自动汇总覆盖率 >= 90%。

### 5.6 权限与组织中心
目标：把 RACI 做成系统授权。
功能点：
1. 组织模型（总部-区域-项目）。
2. 角色模型（GM/DIR/MGR/SPC）。
3. 授权矩阵（Skill 到角色的 RACI 映射）。
4. 越权检测与审计告警。

验收标准：
- 关键 Skill 的唯一 A 约束生效率 100%。

### 5.7 风险与应急中心
目标：问题前置与红线闭环。
功能点：
1. 风险台账与分级。
2. 应急预案库与演练计划。
3. 故障/安全事件战情室。
4. 复盘整改闭环跟踪。

验收标准：
- 重大事件复盘及时率 100%。

### 5.8 预算与供应商中心
目标：把投入产出与外采纳入同一流程。
功能点：
1. 预算编制与滚动调整。
2. 项目到预算到 ROI 的追溯链。
3. 采购流程、合同里程碑、付款控制。
4. 供应商准入、绩效、风险预警。

验收标准：
- 预算申请 Skill 追溯率 100%。

### 5.9 审计与归档中心
目标：满足内审外审要求。
功能点：
1. 全链路操作日志。
2. 证据索引与检索。
3. 审计报表导出。
4. 合规规则检查。

验收标准：
- 核心流程审计证据齐套率 100%。

## 6. 关键流程系统化示例

### 6.1 集团智慧物业平台新建上线（系统链路）
触发：新战略项目申请。

系统自动执行：
1. 创建 Skill 实例链：`1.02.04 -> 3.01.03 -> 3.02.01 -> 3.02.03 -> 3.02.04 -> 3.03.02`。
2. 自动分配任务到 RACI 对应角色。
3. 自动生成模板：立项单、验收单、上线报告、复盘报告。
4. 自动汇总 KPI：里程碑按期率、一次验收通过率、上线故障率。

### 6.2 全网故障突发应急（系统链路）
触发：监控系统告警达到 P1 规则。

系统自动执行：
1. 触发 `9.03.01` 战情流程。
2. 自动升级 GM 并发送统一通知。
3. 自动创建外部沟通任务 `9.03.02`。
4. 24 小时倒计时提醒复盘任务 `9.03.04`。

### 6.3 年度预算编制（系统链路）
触发：预算季开始。

系统自动执行：
1. 拉起 `8.01.01` 流程并收集项目池。
2. 调用历史 ROI 数据填充预算建议。
3. 触发预算评审会签（财务、法务、信息化）。
4. 输出预算书与偏差管控任务。

## 7. 对接清单（最小集成）
1. SSO/统一身份：获取组织与账号。
2. 工单系统：同步任务与处理状态。
3. 监控系统：接收故障告警与 SLA 数据。
4. 安全平台：接收漏洞与事件数据。
5. 财务系统：接收预算、付款、成本数据。
6. BI 平台：展示 KPI 与驾驶舱。

## 8. 非功能要求
1. 可用性：系统可用性 >= 99.9%。
2. 安全性：关键操作双因子；全链路审计。
3. 性能：关键页面 P95 响应 <= 2 秒。
4. 扩展性：Skill 规则可配置，不改代码可新增流程。

## 9. 分期上线计划

### P0（4 周）最小可用版
1. Skill 字典中心。
2. 三条主流程编排。
3. 模板中心基础能力。
4. RACI 权限生效。

### P1（8 周）规模化版
1. KPI 自动汇总。
2. 风险应急中心。
3. 预算与供应商中心。
4. 季度通报自动生成。

### P2（12 周）优化版
1. 智能预警与建议。
2. 跨区域对标分析。
3. AI 场景落地管理。

## 10. 验收 KPI（系统级）
1. Skill 在线化覆盖率 >= 95%。
2. 核心流程自动流转成功率 >= 98%。
3. 模板自动归档率 >= 95%。
4. KPI 自动采集覆盖率 >= 90%。
5. 审计问题闭环时效提升 >= 30%。

## 11. 实施组织建议
1. 产品负责人：信息化 PMO 牵头。
2. 研发负责人：平台研发负责人。
3. 业务 Owner：各条线 DIR。
4. 治理委员会：GM + PMO + 财务 + 安全 + 审计。

## 12. 一句话落地标准
- 不是把 Skill 写在文档里，而是把 Skill 运行在系统里，做到每件事“有流程、有责任、有证据、有指标、有复盘”。
